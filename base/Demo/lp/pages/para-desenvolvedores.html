Para Desenvolvedores

API REST, SDKs, webhooks e documentaēćo completa para integrar seus sistemas ao Zyntra de forma rįpida e segura.

API REST Zyntra

A API REST do Zyntra permite que desenvolvedores integrem qualquer sistema externo ą nossa plataforma. Com endpoints bem documentados e padrões modernos de desenvolvimento, vocź pode automatizar processos, sincronizar dados e construir soluēões customizadas que se conectam ao ERP dos seus clientes.

Nossa API segue os princķpios RESTful, utiliza JSON como formato de dados e autenticaēćo via OAuth 2.0. Todas as chamadas sćo feitas sobre HTTPS com TLS 1.3, garantindo a seguranēa dos dados em trānsito.

Recursos Disponķveis via API

  • Clientes e Fornecedores: CRUD completo de cadastros, incluindo endereēos e contatos
  • Produtos e Serviēos: Gestćo de catįlogo, preēos, estoque e categorias
  • Pedidos de Venda: Criaēćo, consulta, atualizaēćo e cancelamento de pedidos
  • Notas Fiscais: Emissćo, consulta e cancelamento de NF-e, NFS-e e NFC-e
  • Financeiro: Contas a pagar, contas a receber, lanēamentos e conciliaēćo
  • Estoque: Movimentaēões, saldos, reservas e transferźncias entre almoxarifados
  • Relatórios: Geraēćo de relatórios financeiros, comerciais e operacionais

SDKs e Bibliotecas

Para facilitar a integraēćo, oferecemos SDKs oficiais nas linguagens mais populares:

  • Python: pip install zyntra-sdk
  • Node.js: npm install @zyntra/sdk
  • PHP: composer require zyntra/sdk
  • C# / .NET: Disponķvel via NuGet: Zyntra.SDK
  • Java: Disponķvel via Maven Central

Todos os SDKs sćo open source e mantidos pela equipe de engenharia da Zyntra. Contribuiēões da comunidade sćo bem-vindas via GitHub.

Webhooks

Configure webhooks para receber notificaēões em tempo real quando eventos acontecem no Zyntra. Isso permite que seus sistemas reajam automaticamente a aēões como:

  • Novo pedido de venda criado ou atualizado
  • Nota fiscal emitida, autorizada ou cancelada
  • Pagamento recebido ou estornado
  • Produto com estoque abaixo do mķnimo
  • Novo cliente cadastrado
  • Status de cobranēa alterado

Os webhooks sćo enviados via HTTP POST com payload em JSON e incluem assinatura HMAC-SHA256 para validaēćo de autenticidade.

Ambiente de Sandbox

Disponibilizamos um ambiente de sandbox completo para que vocź desenvolva e teste suas integraēões sem risco de afetar dados reais. O sandbox replica todas as funcionalidades da API de produēćo, incluindo emissćo de notas fiscais em ambiente de homologaēćo da SEFAZ.

Para criar sua conta de sandbox, acesse developers.zyntra.com.br e registre-se gratuitamente. Vocź receberį credenciais de API e acesso ą documentaēćo interativa com exemplos em todas as linguagens suportadas.

Rate Limits e Boas Prįticas

  • Plano Profissional: 1.000 requisiēões por minuto
  • Plano Enterprise: 5.000 requisiēões por minuto (customizįvel)
  • Utilize paginaēćo em listagens para otimizar o consumo de recursos
  • Implemente cache local para dados que mudam com pouca frequźncia
  • Utilize webhooks ao invés de polling para monitorar mudanēas
  • Sempre valide a assinatura dos webhooks antes de processar o payload
